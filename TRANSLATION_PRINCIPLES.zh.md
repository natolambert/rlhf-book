# RLHF Book 中文翻译原则

本项目将 `book/` 保留为英文原版，中文译稿放在 `book-zh/`。目标是在不破坏原书 Markdown、公式、引用、图片和构建流程的前提下，生成适合中文科技书阅读和打印的版本。

## 总体风格

- 译文采用正式但不生硬的中文科技书风格，避免机器翻译腔。
- 保持原文的论证顺序、段落层次、标题层级、脚注、引用和图表位置。
- 不擅自增删作者观点。必要的译者说明应明确标记为“译者注”，且默认不加入正文。
- 保留作者在技术判断上的语气，例如“不确定”“经验上”“通常”等限定词。

## 术语策略

- 常见且已有稳定译法的术语使用中文，例如 reinforcement learning 译为“强化学习”，language model 译为“语言模型”。
- 首次出现的重要术语采用“中文（English）”形式，之后优先使用中文简称。
- 缩写和算法名通常保留英文，例如 RLHF、PPO、DPO、GRPO、RLOO、SFT、RLVR、KL。
- 不适合翻译或翻译后更难懂的术语保留英文，必要时在首次出现时解释。
- 数学符号、变量名、函数名、分布名不翻译，例如 `\pi`、policy $\pi_\theta$、reward $r(x, y)$。

## 不翻译或谨慎翻译的内容

- 数学公式、LaTeX 环境、代码块、命令、文件路径、URL、BibTeX key 不翻译。
- 论文标题、书名、模型名、产品名、公司名、数据集名默认保留英文。
- 图片路径和链接锚点不改，避免破坏构建和站内链接。
- 表格中的技术标签、benchmark 名称、模型名称保留英文；说明性文字翻译。
- 代码注释可以在不影响代码语义时翻译；代码本身不改。

## Markdown 和构建约束

- 保留原有标题层级、列表结构、引用语法、图片语法、表格结构、代码围栏和数学分隔符。
- 保留 Pandoc/crossref/citeproc 所需语法，例如 `[@paper]`、`{#fig:name}`、`(#eq:name)`。
- 不引入智能引号、长破折号或不可见空格。中文标点可以使用全角标点，但 Markdown/LaTeX 语法周围保持 ASCII。
- 每个章节翻译完成后，检查代码围栏数量、图片引用数量、引用 key 和数学块是否仍然匹配原文。
